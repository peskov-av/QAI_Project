# AI-Driven QA Pipeline

Курсовая работа: "Проектирование и реализация AI-driven QA-пайплайна для автоматизированного тестирования систем"

## Архитектура
Пайплайн включает следующие этапы:
1. **PII Guardrails**: Проверка входного чек-листа на наличие персональных данных и их маскирование.
2. **Test Design (LLM)**: Генерация JSON-контрактов тест-кейсов на основе требований.
3. **Test Generation (LLM)**: Автоматическая генерация кода автотестов (pytest) из JSON-контрактов.
4. **Code Quality**: Статический анализ сгенерированного кода (flake8).
5. **AI Code Review**: Рецензирование кода с помощью LLM с сохранением артефакта.
6. **Execution**: Запуск тестов и сбор Allure-результатов.
7. **AI Analysis**: Анализ логов прогона, генерация QA-summary и автоматическое формирование баг-репорта (JSON) при обнаружении дефектов.

## Установка
1. Клонируйте репозиторий.
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Установите API ключ Google Gemini:
   ```bash
   export GOOGLE_API_KEY="your_key"
   ```

## Запуск
```bash
python -m qa_pipeline.pipeline
```

## Структура проекта
- `qa_pipeline/`: Основной код пайплайна.
- `test_data/checklists/`: Входные бизнес-требования.
- `generated_tests/`: Сгенерированные файлы тестов.
- `reports/`: Артефакты, логи и Allure-отчеты.
- `.github/workflows/`: CI/CD конфигурация.
